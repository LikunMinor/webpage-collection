# 个人网页收集站

一个纯静态的个人网页收藏首页，适合部署到 GitHub Pages。

## 使用

直接打开 `index.html` 即可查看页面。新增或修改收藏时，编辑 `assets/sites.js` 里的 `window.SITES` 数据，然后提交到 GitHub。

页面支持搜索标题和描述、点击多个标签按交集筛选、分页以及每页数量选择。

## 数据说明

所有收藏数据都维护在 `assets/sites.js` 中，格式是一个站点对象数组：

```js
window.SITES = [
  {
    title: "站点标题",
    url: "https://example.com",
    description: "站点用途说明",
    tags: ["工具", "前端", "AI"],
  },
];
```

每条收藏必须包含下面 4 个字段：

| 字段 | 类型 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| `title` | 字符串 | 是 | 网站名称。页面卡片标题会使用这个字段，并且标题可点击跳转到 `url`。 |
| `url` | 字符串 | 是 | 网站完整链接。建议包含 `https://`，避免部署后跳转异常。 |
| `description` | 字符串 | 是 | 网站用途说明。搜索功能会匹配这个字段。 |
| `tags` | 字符串数组 | 是 | 网站标签。用于标签筛选和标签总数统计。 |

## 数据规则

- 展示顺序按照 `assets/sites.js` 中的数组顺序决定，排在前面的站点会先显示。
- 搜索只匹配 `title` 和 `description`，不会匹配 `url` 或 `tags`。
- 标签筛选支持多选，多个标签同时选中时按交集筛选：站点必须同时包含所有已选标签才会显示。
- 首页顶部的“收录站点”来自 `window.SITES.length`。
- 首页顶部的“标签总数”来自所有 `tags` 去重后的数量。
- 筛选区会渲染全部标签，默认只显示一行，点击“展开标签”后显示全部。
- 单个网站卡片最多展示前 4 个标签；如果 `tags` 超过 4 个，后面的标签仍参与筛选和统计，但不会在卡片里展示。
- 标签名称区分大小写和空格。建议统一写法，例如始终使用 `AI 工具`，不要混用 `AI工具`、`ai 工具`。

## 新增收藏示例

在 `window.SITES` 数组里追加一个对象：

```js
{
  title: "MDN Web Docs",
  url: "https://developer.mozilla.org",
  description: "Web 技术标准文档，覆盖 HTML、CSS、JavaScript 和浏览器 API。",
  tags: ["开发", "文档", "前端", "标准"],
}
```

新增后保存文件并提交到 GitHub。GitHub Pages 部署完成后，页面会显示新数据。
